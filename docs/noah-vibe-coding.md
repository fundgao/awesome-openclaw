# Noah：vibe coding 是伪概念

> 来源：Noah
> 日期：2026-04-10

## 核心观点

**vibe coding 最关键的是"把东西做对"的判断力，而不是"把东西快速做出来"的能力。**

---

## 放弃全自动化

- 以前也很追求 vibe coding 的全自动化
- 很快发现这是一个**伪概念**
- 不是说实现不了，几个月前就可以让 Claude Code 连续工作几个小时了
- 只是存在很多问题，至今仍然存在

---

## 快 vs 对

| 维度 | 描述 |
|------|------|
| **做得快** | 缩短了从想法到结果的距离 |
| **做得对** | 决定这段距离是不是走正确的道路，通往正确的方向 |

vibe coding 的最大优势是**生成速度极快**。

但很多人抱着错误的理念：**认为 vibe coding 是用产出替代思考。**

- 代码自动写
- 页面自动搭
- 功能自动跑

这些都是**实现层面**。

> 即使有一天 AI 能够一秒钟做出一个 linux 系统，也解决不了什么真正的问题。

---

## 真正的问题

现在很多人用 vibe coding 做的产品，**自己都不会用**。

一个好的产品恰恰体现在对细节的把控上。

- 产品自己都不会用 → 不可能面向市场
- 不可能获得大量的用户

---

## 结论

> 不管是写代码还是做别的，但凡对结果质量有要求、有追求，要做真正有价值、可持续的事情，**还得是人**。
> 
> **只有人能够坚持正确。**
