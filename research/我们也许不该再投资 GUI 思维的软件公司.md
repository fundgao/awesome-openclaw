# Agent 正在绕开 GUI：软件竞争的范式转移

文｜真格投资总监 钟天杰

2026 年初，Google Workspace、Salesforce、Atlassian、Obsidian、Polymarket 等一批主流软件公司相继官宣支持和开发 CLI 和官方 MCP 服务。Stripe、Cloudflare、Supabase、Sentry、Vercel 等已有 CLI 的企业主动面向 Agents 提供官方 skills。一个巨大的趋势正在显现：这些公司过去花了数十亿美元打磨 GUI，而现在他们在主动绕开它。

我们应该重新审视人类与软件的关系：Agents 是比人类大十倍的用户群，我们也许不应该再投资带有「界面」思维的软件公司。

---

## 人类是 Agent 的一种形态

人类的每一份工作暗含了一类「长程目标」任务。销售盯成交、采购控成本、法务管风控、市场做传播。这些目标在入职第一天就定义好了，并在工作中不断微调。

每家企业同样可以被抽象成任务协作的集合。Elon Musk 曾有过精准的表述：「Nvidia 的输出是向台积电 FTP 文件。」这体现了他对产业链协作本质的第一性原理描述。

当我们在谈论 Agent 完成任务时，每一次 AI runtime 的运行，都在实例化同样的结构：**一个被分配了长程目标的执行者，调用资源，完成任务。这和人类完成一份工作没有本质区别。**

遗憾的是，人类和 Agent 唯一的区别可能在于硬件——人类是效率更低的 Agent。我们的注意力带宽极窄，工作记忆极浅，需要持续的视觉锚点才能维持任务状态。也正是因为这样的硬件缺陷，在过去几十年催生了一个庞大的软件产业。

---

## GUI 是人类认知缺陷的补丁

**人类是被注意力系统限制的 Agent。** GUI 的存在是一个补偿系统，通过视觉引导、空间布局、即时反馈，帮助一个带宽极度有限的执行者勉强完成被分配的长程目标。

Figma 的精美、Notion 的简洁、Linear 的流畅……这些设计溢价的本质是让人类更愿意盯着屏幕。它们穿着效率工具的外衣，却呈现出和短视频产品类似的底层逻辑。人类对 GUI 的需求，包括「用户体验」行业的繁荣，不是因为 GUI 好，恰恰是围绕人类硬件局限性的商业化。

我们不妨称之为一种**界面税**，是一种为人类认知缺陷支付的补偿成本。那些被归入效率工具类目的软件产品，实则享受的是注意力经济的红利。以用户体验为护城河的软件公司，都是注意力公司的变体。**一旦用户不是人类，这种竞争优势就会归零。**

Agent 没有这个缺陷，它推翻了整个前提。

---

## 站在 Agent 执行长程任务的必经节点上

每个 Agent 被实例化时，人类需要给它一个任务定义。这个目标跨越多个软件、多个步骤、多个时间节点，是项目级的，甚至是终身使命。

Agent 至少需要四样东西来完成一个最基础的任务：

- **信息与数据**是现实世界和 Agent 之间的桥梁。拥有高信息密度的独占数据，价值反而会随着计算能力的降本而上升。
- **执行权限**是 Agent 意图和现实改变之间的桥梁。Agent 需要真正改变世界的能力：发送邮件、修改代码、转移资金、下达订单。
- **可信度**是 Agent 身份和社会系统之间的桥梁。Agent 代表其背后实体的信用背书和风险转移。
- **算力与模型**是 Agent 思考和行动之间的桥梁，持续消耗推理资源以完成规划和判断。

这四者都不在软件「里面」，而在软件「边界上」。Agent 时代的软件，要主动把自己暴露出去、让 Agent 路过时必须调用。**站在 Agent 的路径上，在每一个长程目标的必经节点上收税，是 Agent 时代的目标。**

这意味着软件竞争方向的反转：旧逻辑是建一个封闭空间，让用户走进来，用体验把他留住。**新逻辑是把自己暴露出去，站在 Agent 执行长程目标的路径上，让它经过时不得不调用你。**

作为创业公司，能定义清楚把自己卡在 Agent 长程任务的哪个节点上，是新时代唯一重要的事。

---

## 协议即软件：Software as Protocols

协议完全不在乎界面是否好看；协议需要稳定、可靠、无处不在。产品竞争靠更好看、更好用、更有品牌溢价。协议竞争靠更早成为标准——**一旦成为某个 Agent 工作流里的默认调用，后来者便很难替代你。谁成为 Agent 的默认工具，谁就将捕获更多的价值。**

让我们把对人类的评价系统同样应用到新时代的产品评价中：
- 接口是否稳定，是一种信用
- 文档是否清晰，是一种美
- 响应是否可预测，是一种靠谱
- 数据是否具有独家性，是一种地位

目前对协议本身的评价体系几乎还是一片空白。我们仍然只能依靠人类能感知的环境（如社交媒体的传播热度）来评价一个协议是否靠谱，这显然需要改变。

**最有价值的软件公司，是那些成为 Agent 工作流里默认协议的公司。**

---

## 不要再开发仅存在于执行层的 GUI 产品

**GUI 在 Agent 时代的价值是满足人类对长程目标运行的掌控感（一种情绪价值）**——从执行工具降级成为控制面板。它仍然有价值，但会变成标配，不构成护城河。大幅降低的研发成本让 GUI 产品变成了一种内容产品，和生产一条短视频、一篇图文笔记没有本质区别。

回看移动互联网时代，同时维护 PC 版和 Mobile 版的团队，往往两个都做不好。现在完全一样。同时维护 GUI 和 Agent-Native 接口，是一种资源的浪费。把核心资源投在 CLI、API、官方 Skills 上——你真正的用户在那里，那个世界即将山呼海啸而来。

在 Agent 原生的网络里，一切才刚刚开始。让我们抛开人类商业文明至今所有以人类作为主力执行者的执念，重新思考所有的组织形态，重新思考每一种生产任务、协作任务、娱乐任务的最佳实践路径。
